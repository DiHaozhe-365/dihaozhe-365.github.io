第 1 章 Java 入门

1.1 Java 的地位

Java 具有 面向对象 、跨平台性 、安全 、稳定 和 多线程 等优良特性，是目前软件设计中优秀的编程语言。Java 不仅可以用来开发大型的应用程序，而且特别适合于 Internet 应用的开发。Java 确实具备了“一旦写成处处可用”的特点，这也是 Java 最初风靡全球的主要原因。Java 是一门正在被广泛使用的编程语言，而且许多新的技术领域都涉及了 Java 语言，Java 已成为网络时代最重要的编程语言之一。

1.2 Java 的特点

Java 是目前使用最为广泛的网络编程语言之一，它具有语法简单、面向对象、稳定、跨平台性、多线程、动态等特点。

相对于 C++ 语言而言，Java 更加简单，因为 C++ 中许多容易混淆的概念或者被 Java 弃之不用了，或者以一种更清楚、更容易理解的方式实现，例如 java 中不再有指针的概念。

基于对象的编程更符合人的思维模式，使人们更容易解决复杂的问题。Java 是面向对象的编程语言，所有代码都是通过创建对象来实现的。Java 支持封装、继承和多态等面向对象的特性，有助于构建可靠、灵活和可复用的代码。

Java 的最重要的特点之一是它的跨平台性。Java 程序可以在不同的操作系统上运行，只需在目标平台上安装 Java 虚拟机（JVM）即可。这使得 Java 成为开发跨平台应用和 Web 服务的理想选择。

Java 允许程序同时执行多个线程，每个线程独立执行不同的任务，提高了程序的并发性和性能。

Java 拥有庞大的社区和丰富的开发资源。有大量的开源框架、库和工具可供选择，例如 Spring、Hibernate、JUnit 等，使开发人员能够更快速、高效地构建各种应用程序。

1.3 Java 技术体系平台介绍

支持面向桌面级应用（如 Windows 下的应用程序）的 Java 平台，即定位个人计算机的应用开发。

包括用户界面接口 AWT 及 Swing，网络功能与国际化、图像处理能力以及输入输出支持等。

此版本以前称为 J2SE。

为开发企业环境下的应用程序提供的一套解决方案，即定位在服务器端的 Web 应用开发。

Java EE 是 Java SE 的扩展，增加了用于服务器开发的类库。如：Servlet 能够延伸服务器的功能，通过请求-响应的模式来处理客户端的请求；JSP 是一种可以将Java 程序代码内嵌在网页内的技术。

此版本以前称为 J2EE。

支持 Java 程序运行在移动终端（手机、机顶盒）上的平台，即定位在消费性电子产品的应用开发。

Java ME 是 Java SE 的内伸，精简了 Java SE 的核心类库，同时也提供自己的扩展类。增加了适合微小装置的类库等。

此版本以前称为 J2ME。

1.4 Java 开发环境搭建

什么是 JDK、JRE、JVM

JDK、JRE 和 JVM 是 Java 开发和运行环境的不同组成部分。

JDK（Java Development Kit）是 Java 开发工具包，它包含了完整的 Java 开发环境。

JDK 提供了用于编译、调试和运行 Java 程序的工具、编译器（javac）、运行时环境（JRE）、开发文档、示例代码和其他相关工具。

JDK 适用于开发人员，用于创建 Java 应用程序、Java 库和 Java Applet 等。

JRE（Java Runtime Environment）是 Java 运行时环境，它是 JDK 的一个子集。

JRE 提供了运行 Java 应用程序所需的所有运行时支持，包括 Java 虚拟机（JVM）、Java 类库和其他必要的文件。

JRE 适用于终端用户，用于执行 Java 程序，而无需进行开发。

JVM（Java Virtual Machine）是 Java 虚拟机，它是 Java 程序的运行环境。

JVM 是 Java 语言的核心概念之一，它提供了一个虚拟的计算机环境，可以在不同的操作系统上执行 Java 字节码。JVM 负责将 Java 字节码翻译成特定操作系统所能理解的机器语言，并管理程序的内存、线程和其他运行时资源。

JDK 包含了 JRE，JRE 包含了 JVM。开发 Java 应用程序时，需要使用 JDK 来编译和调试程序，而最终运行 Java 程序时，只需 JRE 即可。

JVM 是 Java 程序的执行引擎，负责将字节码转化为机器码，并执行程序。

总结起来，JDK 提供了完整的 Java 开发环境，JRE 提供了运行 Java 程序所需的运行时环境，而 JVM 则是具体执行 Java 程序的运行引擎。它们共同构成了 Java 的开发和运行平台。

JDK 版本选择

自 Java 8 版本发布以来，其后的每次更新，都会有小伙伴高呼：Java 8 YYDS!

此外，某美国软件开发商在对近千名专业的 Java 开发者调研后，发布的《2022 年 Java 开发者生产力报告》称：八年前发布的 Java 8 依然是 Java 中应用最广泛的版本，占比 37%，其次是 Java 11，占比 29%。

在 Java 17 正式发布之前，Java 开发框架 Spring 率先宣布，Spring Framework 6 和 Spring Boot 3 计划在 2022 年第四季度实现总体可用性的高端基线：

1.5 Java 程序的开发步骤

Java 程序开发三步骤：编写、编译、运行。

  • 将 Java 代码编写到扩展名为 .java 的源文件中。
  • 通过 javac.exe 命令对该 .java 文件进行编译，生成一个或多个字节码文件。
  • 通过 java.exe 命令对生成的 .class 文件进行运行。

    • 一个源文件中可以有多个类，编译后会生成多个 .class 字节码文件。
    • 但是一个源文件只能有一个 public 的类。

1.7 Java 反编译

Java 反编译是指将已经编译成字节码的 Java 程序文件（.class 文件）转换回可读性较高的 Java 源代码文件（.java 文件）。这个过程称为反编译。

有时候，我们可能需要查看或修改某个 Java 程序的源代码，但却无法获取原始的 .java 源文件，这时就可以使用反编译工具将字节码文件转换回可读的 Java 源代码。

在 Java 中，常用的反编译工具有 JD-GUI、Fernflower、Procyon、JAD 等。这些工具可以将 .class 文件转换为 Java 源代码，使我们能够查看类的代码实现、理解程序的功能，或进行必要的修改。

有些 Java 程序可能经过特殊的编译和混淆技术，使得反编译变得更加困难。此外，反编译行为也必须符合法律规定，不能用于非法用途。因此，在使用反编译工具时，需要遵循合法和道德的原则，并确保自己对程序的使用是合法和合规的。

1.8 Java 之父——James Gosling

  • 詹姆斯 · 高斯林（James Gosling）先生以“Java 技术之父”而闻名于世。他是 Java 的创始人，亲手设计了 Java 语言，并开发了 Java 编译器和 Java 虚拟机，使 Java 成为了世界上最流行的开发语言。
  • 1984 年，詹姆斯 · 高斯林加入 Sun 公司，并一直服务于 Sun 公司，直至 2010 年前后，Sun 被 Oracle 并购而加入 Oracle，担任客户端软件集团的首席技术官。
  • 2010 年 4 月，詹姆斯 · 高斯林从 Oracle 离职。
  • 2011 年 3 月 29 日，高斯林在个人博客上宣布将加入 Google。
  • 2011 年 8 月 30 日，仅仅加入 Google 数月之后的高斯林就在个人博客上宣布离开 Google，加盟一家从事海洋机器人研究的创业公司 Liquid Robotics，担任首席软件架构师。
  • 2011 年 5 月建立的 Scala 公司 Typesafe Inc.， 高斯林被聘请为公司顾问。
